redtiger level1 writeup

отправлено 2015-12-08 | 修改于 2015-12-13   |   в writeup   |  

这个系列的题目都是考察sql注入相关知识的，在这一系列的writeup中，我不仅仅会给出详细的解题思路，还会由浅入深的分析所用到的sql注入的知识。
打开第一关地址：https://redtiger.labs.overthewire.org/level1.php，我们看到如下页面：

在这里，我们知道了我们需要的数据在表level1_users中。我们点击Category，看到页面发生了变化：

故而这一关的注入点便是cat参数，我们假设当前使用的表是cats，那么我们可以大体构造出页面后台所使用的sql查询语句为：select * from cats where cat = 1 --。接下来，我们使用order by获取字段数，具体操作我们可以提交cat参数为1 order by 4，即构造如下链接：https://redtiger.labs.overthewire.org/level1.php?cat=1%20order%20by%204–，此时我们在构造一下后台数据库查询语句为：select * from cats where cat = 1 order by 4 --，访问页面后我们发现页面正常，而此时如果我们将参数设置为’1 order by 5 –’，访问页面发现页面出错，出现如下提示：

这说明在后台的sql查询语句中对4个字段进行了查询，即其sql语句为：select 1,2,3,4 from cats where cat = 1 order by 5 --。接下来我们继续使用union select来获取数据，由于该页面后台对4个字段进行查询，而union select语句的作用是将两次查询的结果合并输出并要求两次查询的字段数必须相同，因此我们构造参数为-1 union select 1,2,3,4 –，此时其后台sql查询语句可以构造如下：select 1,2,3,4 from cats where cat = -1 union select 1,2,3,4 --，此时前面的查询由于参数cat的值为-1，查询结果为空，而后面的union查询输出1，2，3，4，故最终我们会看到如下页面：

这样，3、4处就是我们后面union select输出的地方，我们尝试着在3处输入version()，然后查询出的数据库版本信息就会显示在页面中3的地方：

这样，我们直接将参数设置为：-1 union select 1,2,username,password from level1_users limit 0,1 –，可以查询出用户名密码的信息了。
